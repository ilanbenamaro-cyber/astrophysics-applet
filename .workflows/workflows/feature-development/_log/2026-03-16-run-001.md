## Non-Obvious Decisions

1. **UV normalization**: Normalized UV coordinates so Earth's diameter (2 × R_earth) maps
   to N/2 image pixels. This is a synthetic choice (real EHT uses wavelength units) but
   is correct for educational simulation — the longest baseline fills the Nyquist frequency.

2. **Conjugate symmetry**: Both (u,v) and (-u,-v) are added per baseline per HA step.
   Required for the IFFT output to be real-valued. Standard radio astronomy property of
   a real brightness distribution's Fourier transform.

3. **Debounce**: `runReconstruction` is debounced at 50ms. Discovered mid-implement that
   `loadPresets` calls `addTelescope` 6× in rapid succession, each triggering the
   onChange callback. Without debounce, 6+ FFT jobs queue simultaneously.

4. **setTimeout(reconstruct, 10)**: The inner FFT computation is deferred 10ms after
   the "Reconstructing…" status message is set, so the status renders before the main
   thread is blocked by FFT work.

5. **grayscaleToCanvas auto-normalize**: Uses actual data min/max, not a fixed [0,255]
   clamp. With sparse UV coverage, the IFFT output amplitudes are much smaller than
   the original pixel values — clamping would produce a nearly-black image.
