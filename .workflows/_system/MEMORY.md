## How This File Works

Entries here are things that are true across projects — not specific to
one codebase. Examples:
- A library that consistently causes problems
- A pattern that sounds good but always creates issues later
- A prompt structure that gets better results from Claude
- A debugging approach that works reliably

If something is project-specific, it belongs in _knowledge/gotchas.md instead.
